# Model-Specific Software
Most VDRs may have a specific type of software to conduct analysis on the data. Those for different VDR models discussed in the [VDR Background](/Background/VDR/Requirements.md) section are:

| Name | Expected Data Format / Type | Software Name | Software Link |
|------|-----------------------------|---------------|---------------|
| Danelec Marine DM100 VDR | Proprietary binary container (audio, radar, NMEA 0183/2000, AIS multiplexed) | Danelec Replay / DM Investigator | link here |
| AMI Marine Float-Free VDR (L3 / L3Harris) | Proprietary binary container (bridge audio, radar, ECDIS, NMEA) | AMI VDR Replay / Investigator | link here |
| JRC JCY-1900 VDR | JRC proprietary VDR format (radar, ECDIS, NMEA, audio) | JRC VDR Player / Investigator | link here |
| IMO-Compliant Float-Free VDR (Generic) | Manufacturer-specific proprietary capsule format | Vendor-specific replay tools (Danelec / NetWave / AMI / Furuno / Consilium) | Varies by manufacturer |
| Yacht Devices VDR (YDVR-04N / YDVR-02) | Raw NMEA 2000 CAN frames stored in .DAT | YDVR Converter, ydn2kconvert, CAN tools | link here |
| Kelvin Hughes MantaDigital VDR | Proprietary Manta container (radar, ECDIS, audio, NMEA) | Manta Replay / Investigator Suite | link here |
| Furuno VR-3000 Series | Furuno proprietary VDR format (radar, ECDIS, audio, NMEA) | VR3000 Replay Software | link here |
| NetWave Float-Free VDR | NetWave proprietary capsule format | NetWave Replay Suite | link here |
| Consilium VDR (S-VDR / FDR) | Consilium proprietary container | Consilium Replay Software | link here |
